OpenSearch Earth Observation Extension Conformance Test Suite

Scope

This executable test suite (ETS) verifies conformance of OpenSearch Description documents with respect to the specifications:

  • OGC OpenSearch Earth Observation Extension

Several conformance classes are defined and test subject is checked against the following specifications:

  • OpenSearch Earth Observation Extension (OGC 13-026)
  • The Atom Syndication Format (RFC-4287)

The following conformance classes are defined:

  • Core
    • The server generates a valid OpenSearch description document.
    • The server defines an URL template for the Atom response type.
    • Verify that the server uses the EO extension namespaces.
    • Verify that the server response is a valid ATOM document.
    • Verify that the server accepts search parameters as KVP.
    • Verify that the server accepts search parameters from the EO extension.
  • Earth Observation Foreign Element : Verify that the server response includes a eop:EarthObservation element for each entry.
  • Parameter range suggestion : The server implements a suggestion of parameters range for the Earth Observation extension elements on the OpenSearch Description document.
  • INSPIRE : The server supports the INSPIRE relevant OpenSearch parameters (queryables).

See the Javadoc documentation for more detailed information about the test classes and test methods that comprise the suite.

How to run the tests

The test suite may be run in any of the following environments:

  • TEAM-Engine: Run the CTL script located in the scripts/osxeo/ directory.

The test run arguments are summarized in Table 1. The Obligation descriptor can have the following values: M (mandatory), O (optional), or C (conditional).

Table 1 - Test run arguments
Name Value domain Obligation Description
iut URI M A URI that refers to the OpenSearch Description document implementation under test.
